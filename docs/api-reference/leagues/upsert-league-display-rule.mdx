---
title: "Upsert A League Display Rule"
description: "Create or update an informational display rule for a league game."
openapi: "POST /leagues/{identifier}/display-rules/upsert"
---

## Permission Required

This endpoint requires a **league-operator API key bound to this league** with the `display_rules` league capability, plus the `leagueOperatorApi` plan feature (api_pro or higher). A free key receives `403 error_api_feature_required`.

## Access / Membership Rules

You must have a **MANAGER or higher** role in the league (or be a TeamBattles staff member) to manage display rules. If you lack the required access, you'll receive a `403` error.

The `{identifier}` path parameter is the league **slug** - the URL-friendly league name (e.g., `my-league`).

## What's Returned

Creates or updates an informational display rule for one of the league's games. On success the response is `{ "success": true, "displayRuleId": "<id>", "timestamp": "<ISO 8601>" }`.

## Request Body

| Parameter          | Type    | Required | Description                                                                       |
| ------------------ | ------- | -------- | -------------------------------------------------------------------------------- |
| `gameId`           | string  | Yes      | Game ID whose display rule should change.                                        |
| `category`         | string  | Yes      | Display rule category (1-100 characters).                                        |
| `categoryOrder`    | number  | Yes      | Sort position of the category among the game's display rules.                    |
| `isCustomCategory` | boolean | Yes      | Whether this is a custom (operator-defined) category rather than a built-in one. |
| `isOverride`       | boolean | Yes      | Whether this rule overrides a default category's content.                        |
| `content`          | object  | Yes      | Rule content: `{ sections: [{ heading?, items: string[] }] }`.                   |
| `scope`            | object  | No       | Optional scoping: `{ modes?: string[], playlists?: string[] }`.                  |
| `categoryIcon`     | string  | No       | Optional icon identifier for the category.                                       |

<RequestExample>

```bash cURL
curl -X POST https://teambattles.gg/api/v1/leagues/my-league/display-rules/upsert \
  -H "Authorization: Bearer tb_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "gameId": "jh7game123",
    "category": "Match Conduct",
    "categoryOrder": 0,
    "isCustomCategory": true,
    "isOverride": false,
    "content": { "sections": [{ "heading": "Sportsmanship", "items": ["Be respectful in chat."] }] }
  }'
```

```javascript JavaScript
const leagueSlug = "my-league";

const response = await fetch(
	`https://teambattles.gg/api/v1/leagues/${leagueSlug}/display-rules/upsert`,
	{
		method: "POST",
		headers: {
			Authorization: "Bearer tb_your_api_key",
			"Content-Type": "application/json"
		},
		body: JSON.stringify({
			gameId: "jh7game123",
			category: "Match Conduct",
			categoryOrder: 0,
			isCustomCategory: true,
			isOverride: false,
			content: {
				sections: [{ heading: "Sportsmanship", items: ["Be respectful in chat."] }]
			}
		})
	}
);

const { success, displayRuleId, timestamp } = await response.json();
console.log(success, displayRuleId, timestamp);
```

```python Python
import requests

league_slug = 'my-league'

response = requests.post(
    f'https://teambattles.gg/api/v1/leagues/{league_slug}/display-rules/upsert',
    headers={'Authorization': 'Bearer tb_your_api_key'},
    json={
        'gameId': 'jh7game123',
        'category': 'Match Conduct',
        'categoryOrder': 0,
        'isCustomCategory': True,
        'isOverride': False,
        'content': {
            'sections': [{'heading': 'Sportsmanship', 'items': ['Be respectful in chat.']}]
        }
    }
)

print(response.json())
```
