## Overview

TeamBattles supports three score submission flows, which differ in authentication,
endpoint, and confirmation behavior:

1. **Captain Score Submission** - a team captain submits on `POST /api/v1/matches/{matchId}/scores`; the opposing captain must confirm.
2. **Game Developer Score Submission** - a game server submits verified scores on `POST /api/v1/game/matches/{matchId}/scores`; scores are auto-confirmed.
3. **League Operator Score Submission** - a league operator submits verified scores on `POST /api/v1/leagues/{identifier}/matches/{matchId}/scores`; scores are auto-confirmed.

## Captain Score Submission

Captains submit scores for their team. The opposing team must confirm the score.

### Submit Score

```bash
POST /api/v1/matches/{matchId}/scores
```

**Headers:**

```
Authorization: Bearer tb_your_api_key
Content-Type: application/json
```

**Body:**

```json
{
	"mapIndex": 0,
	"mapId": "dust2",
	"creatorTeamScore": 13,
	"opponentTeamScore": 7,
	"screenshotUrls": ["https://example.com/screenshot.png"]
}
```

| Field               | Type     | Required | Description                  |
| ------------------- | -------- | -------- | ---------------------------- |
| `mapIndex`          | number   | Yes      | Zero-based map index         |
| `mapId`             | string   | Yes      | Map identifier string        |
| `creatorTeamScore`  | number   | Yes      | Creator team's score (>= 0)  |
| `opponentTeamScore` | number   | Yes      | Accepted team's score (>= 0) |
| `screenshotUrls`    | string[] | No       | Array of screenshot URLs     |

**Response:**

```json
{
	"success": true,
	"action": "submitted",
	"mapIndex": 0,
	"scoreStatus": "PENDING"
}
```

### Confirm Score

The opposing team's captain confirms a submitted score using PATCH on the same endpoint.

```bash
PATCH /api/v1/matches/{matchId}/scores
```

**Body:**

```json
{
	"mapIndex": 0
}
```

**Response:**

```json
{
	"success": true,
	"action": "confirmed",
	"mapIndex": 0,
	"scoreStatus": "CONFIRMED"
}
```

### Get Scores

Retrieve all map scores for a match, including series score summary.

```bash
GET /api/v1/matches/{matchId}/scores
```

**Response:**

```json
{
	"scores": [
		{
			"mapIndex": 0,
			"mapId": "dust2",
			"creatorTeamScore": 13,
			"opponentTeamScore": 7,
			"screenshotUrls": [],
			"scoreStatus": "CONFIRMED",
			"submittedBy": { "userId": "...", "teamId": "...", "name": "Player" },
			"confirmedBy": { "userId": "...", "teamId": "...", "confirmedAt": "..." },
			"createdAt": "2024-01-15T12:00:00.000Z"
		}
	],
	"seriesScore": {
		"creatorMapWins": 1,
		"opponentMapWins": 0
	},
	"timestamp": "2024-01-15T12:30:00.000Z"
}
```

## Game Developer Score Submission

<Note>
	Requires Game Developer approval for the specific game. Scores submitted via this endpoint are
	automatically confirmed.
</Note>

### Submit Verified Score

```bash
POST /api/v1/game/matches/{matchId}/scores
```

**Headers:**

```
Authorization: Bearer tb_your_game_api_key
Content-Type: application/json
```

**Body:**

```json
{
	"maps": [
		{
			"mapIndex": 0,
			"mapId": "de_dust2",
			"creatorTeamScore": 13,
			"opponentTeamScore": 9,
			"screenshotUrls": ["https://..."],
			"playerStats": {}
		}
	]
}
```

**Response:**

```json
{
	"success": true,
	"submitted": [{ "mapIndex": 0, "status": "confirmed" }],
	"count": 1
}
```

`success` is `true` only when every map confirms; `count` is the number of confirmed maps. A failed map appears as `{ "mapIndex": n, "status": "failed", "error": "<code>" }` and does not abort the others. The per-item `status` is lowercase.

### Submit Single Map Score

```bash
POST /api/v1/game/matches/{matchId}/scores/{mapIndex}
```

Submit one map. `mapIndex` comes from the URL path, so it is not in the body.

**Body:**

```json
{
	"mapId": "de_dust2",
	"creatorTeamScore": 13,
	"opponentTeamScore": 9
}
```

**Response:**

```json
{
	"success": true,
	"mapIndex": 0,
	"scoreStatus": "CONFIRMED"
}
```

(`scoreStatus` is uppercase `CONFIRMED` here, unlike the lowercase batch `status`.)

### Batch Match Scores

Submit map scores for **multiple matches** in one request. Each item is independently scope-checked and processed; the request always returns `200` with a per-item result, so a single bad item does not fail the whole batch.

```bash
POST /api/v1/game/matches/batch-scores
```

**Headers:**

```
Authorization: Bearer tb_your_game_api_key
Content-Type: application/json
```

Batch submission is naturally idempotent (re-submitting an unconfirmed map overwrites it; a confirmed map returns `error_score_already_submitted_by_game`), so it does not use an `Idempotency-Key`.

**Body:** an `items` array (1-50 items). Each item:

| Field               | Type     | Required | Description                  |
| ------------------- | -------- | -------- | ---------------------------- |
| `matchId`           | string   | Yes      | Match to submit for          |
| `mapIndex`          | number   | Yes      | Zero-based map index (>= 0)  |
| `mapId`             | string   | Yes      | Map identifier               |
| `creatorTeamScore`  | number   | Yes      | Creator team score (>= 0)    |
| `opponentTeamScore` | number   | Yes      | Accepted team score (>= 0)   |
| `screenshotUrls`    | string[] | No       | Screenshot URLs              |
| `playerStats`       | object   | No       | Per-player stats for the map |

```json
{
	"items": [
		{ "matchId": "match_123", "mapIndex": 0, "mapId": "dust2", "creatorTeamScore": 13, "opponentTeamScore": 7 },
		{ "matchId": "match_456", "mapIndex": 1, "mapId": "mirage", "creatorTeamScore": 10, "opponentTeamScore": 13 }
	]
}
```

**Response (always 200):**

```json
{
	"success": true,
	"submitted": [
		{ "matchId": "match_123", "mapIndex": 0, "status": "confirmed" },
		{ "matchId": "match_456", "mapIndex": 1, "status": "failed", "error": "error_game_scope_mismatch" }
	],
	"count": 1
}
```

`success` is `true` only when every item is `confirmed`; `count` is the number of confirmed items. A failed item carries a bare machine `error` code (an item for a match outside your bound game yields `error_game_scope_mismatch`). A body with more than 50 items is rejected with `400 error_invalid_num_items`.

Requires `game.scores: write`.

## League Operator Score Submission

<Note>
	League operators submit verified scores with a league-operator API key. Scores
	auto-confirm immediately - the opposing captain does not need to confirm. The key
	must be bound to the league whose match you are scoring.
</Note>

### Submit Map Scores

```bash
POST /api/v1/leagues/{identifier}/matches/{matchId}/scores
```

Submit every map for the match in one request (same envelope as the game-developer batch).

**Body:**

```json
{
	"maps": [
		{ "mapIndex": 0, "mapId": "de_dust2", "creatorTeamScore": 13, "opponentTeamScore": 9 }
	]
}
```

**Response:**

```json
{
	"success": true,
	"submitted": [{ "mapIndex": 0, "status": "confirmed" }],
	"count": 1
}
```

### Submit Single Map Score

```bash
POST /api/v1/leagues/{identifier}/matches/{matchId}/scores/{mapIndex}
```

Submit one map. `mapIndex` comes from the URL path, so it is not in the body.

**Body:**

```json
{
	"mapId": "de_dust2",
	"creatorTeamScore": 13,
	"opponentTeamScore": 9
}
```

**Response:**

```json
{
	"success": true,
	"mapIndex": 0,
	"scoreStatus": "CONFIRMED"
}
```

This flow requires the league `scores` capability (`leagueCapability: scores`) and the
`leagueOperatorApi` feature, which is gated on the league owner's plan.

## Key Differences

| Feature             | Captain Submission                 | Game Developer Submission | League Operator              |
| ------------------- | ---------------------------------- | ------------------------- | ---------------------------- |
| Authentication      | User API key                       | Game developer API key    | League-operator API key      |
| Auto-confirmation   | No (requires opponent)             | Yes                       | Yes                          |
| Metadata support    | No                                 | Yes                       | Yes                          |
| Game scope          | Any                                | Approved games only       | Bound league only            |
| Permission required | `matches.team_matches: read-write` | `game.scores: write`      | `leagueCapability: scores` (feature-gated `leagueOperatorApi`) |
