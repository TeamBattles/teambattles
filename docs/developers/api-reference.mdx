## Pagination

The v1 API is mid-migration and currently exposes two pagination envelope shapes. Check each endpoint's reference page for its exact response shape.

### Native envelope (match listing endpoints)

The three match listing endpoints use Convex's native pagination envelope:

- `POST /api/v1/user/matches` - see [User Matches](/api-reference/user/matches)
- `POST /api/v1/teams/{teamId}/matches` - see [Team Matches](/api-reference/teams/matches)
- `POST /api/v1/orgs/{orgId}/matches` - see [Organization Matches](/api-reference/orgs/matches)

**Request fields:** `numItems` (page size, default 25) and `cursor` (opaque string from the previous response, or omit/`null` for the first page).

**Response shape:** `{ page, isDone, continueCursor }`. Iterate `page` for results, check `isDone` to know when you've reached the end, and pass `continueCursor` as `cursor` in the next request.

```bash
POST /api/v1/user/matches
Content-Type: application/json

{
  "numItems": 25,
  "cursor": "g2wAAAACYQJhAmo="
}
```

### Legacy envelope (all other v1 endpoints)

Most remaining v1 paginated endpoints still wrap results in a legacy envelope: `{ matches, pagination: { cursor, hasMore }, timestamp }`. Pass `cursor` (and typically `limit`) in the request body to get the next page. This shape will be migrated to the native envelope in a follow-up release.

```bash
POST /api/v1/game/matches
Content-Type: application/json

{
  "cursor": "25",
  "limit": 25
}
```

## Request limits

| Code                      | HTTP | Meaning                                                                                                                          |
| ------------------------- | ---- | ------------------------------------------------------------------------------------------------------------------------------- |
| `error_invalid_num_items` | 400  | Item count out of range: the `numItems` 1-100 range on match-listing endpoints, or the 50-item ceiling on score-batch endpoints. |
| `error_invalid_limit`     | 400  | The `limit` is out of range: 1-50 on the league discovery list (`POST /leagues`), 1-100 on every other paginated endpoint (including league standings and penalties).                                                     |

For `Idempotency-Key` conflicts (`409 error_idempotency_key_conflict`) see the
[Idempotency](/developers/idempotency) guide.
